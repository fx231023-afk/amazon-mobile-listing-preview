# Amazon 手机端预览工具 - 线上部署说明

## 团队日常使用方式

部署完成后，团队成员只需要打开同一个网页地址：

```text
https://你的应用地址
```

使用流程：

1. 打开网页。
2. 上传主图 / 副图 / A+ 图片。
3. 调整商品标题、价格、五点描述等信息。
4. 点击“生成 24 小时分享链接”。
5. 把生成的链接发给手机或同事查看。

同事不需要安装 Node.js，也不需要运行任何脚本。

## 分享链接规则

- 分享链接有效期：24 小时。
- 图片只用于临时预览。
- 到期后服务会自动清理分享数据。
- 如果服务器重启或免费平台休眠，临时链接可能提前失效；正式团队使用建议选择稳定付费实例。

## Render 部署方式

1. 把项目上传到 GitHub。
2. 打开 Render，选择 New Web Service。
3. 连接这个 GitHub 项目。
4. Render 会自动读取 `render.yaml`。
5. 部署完成后，Render 会给你一个固定网址。

如果手动填写：

```text
Build Command: npm install && npm run build
Start Command: npm start
```

环境变量：

```text
NODE_ENV=production
MAX_BODY_MB=80
```

## 普通云服务器部署方式

服务器安装 Node.js 20 或以上版本后，在项目目录运行：

```bash
npm install
npm run build
npm start
```

默认端口是 `4173`。

如果需要改端口：

```bash
PORT=3000 npm start
```

Windows PowerShell：

```powershell
$env:PORT=3000
npm start
```

## 数据保存位置

临时分享数据默认保存到：

```text
temp-shares
```

截图导出服务器备份默认保存到：

```text
exports
```

可以用环境变量修改：

```text
SHARES_DIR=/data/temp-shares
EXPORTS_DIR=/data/exports
```

## 后期费用

这个应用本身没有接口调用费，也没有数据库费用。

可能产生费用的是：

- 服务器或部署平台月费
- 图片临时存储占用的磁盘空间
- 团队访问产生的流量

如果只是小团队日常预览，费用通常很低；如果要稳定给多人使用，建议不要依赖会休眠的免费实例。
